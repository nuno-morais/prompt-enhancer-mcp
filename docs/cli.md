# Prompt‑Enhancer MCP – CLI (`mcp`)

The CLI provides a fast, one‑line way to optimise prompt drafts locally using your Ollama model.

## Installation

```bash
npm i -g prompt‑enhancer-mcp   # installs the `mcp` binary globally
```

The package ships a **bin** entry that points to `dist/bin/mcp.js`. After `npm run build` the compiled file will be generated automatically.

## Usage

### Pipe a draft into the CLI

```bash
echo "Write a concise summary of the project plan." | mcp
```

### Provide the draft as an argument

```bash
mcp --draft "I need a short, friendly reminder about my meeting tomorrow."
```

### Enable brainstorming mode

```bash
mcp --draft "Ideas for a weekend hackathon." --brainstorm
```

### Include an explanatory note

```bash
mcp --draft "Summarise the repository README." --explain
```

### Target a specific model format

```bash
mcp --draft "Generate a prompt for Claude." --target claude
```

### Override the Ollama model name

```bash
mcp --draft "Create a prompt for a coding assistant." --model llama3.1
```

## Options Overview

| Flag | Description |
|------|-------------|
| `-d, --draft <draft>` | Prompt draft string. If omitted, the CLI reads from **stdin**. |
| `-b, --brainstorm` | Enables multi‑persona brainstorming (via the `brainstorm` parameter). |
| `-e, --explain` | Returns an extra one‑line summary of the changes made. |
| `-t, --target <model>` | Desired target model (`generic`, `claude`, `gpt4o`, `gemini`). |
| `-m, --model <ollama>` | Override the Ollama model used for optimisation. |
| `-h, --help` | Show help information (auto‑generated by **commander**). |

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success – optimized prompt(s) written to `stdout`. |
| `1` | Invalid usage (e.g., missing draft). |
| `2` | Internal error (network, Ollama failure, etc.). |

## Example Output

```text
$ echo "Give a detailed explanation of quantum entanglement." | mcp --explain
Optimized prompt: Explain quantum entanglement in ≤ 150 tokens, focusing on core concepts and avoiding jargon.

Explanation: Removed redundant phrasing and tightened the request to a concrete token budget.
```

## Development

After cloning the repo:

```bash
npm install          # installs dependencies (including commander & cosmiconfig)
npm run build        # compiles TypeScript to `dist/`
npm start            # runs the server (for API mode)
```

The CLI source lives at `src/bin/mcp.ts`. When you run `npm run build`, the compiled file appears at `dist/bin/mcp.js` and is exposed via the `bin` field in `package.json`.
