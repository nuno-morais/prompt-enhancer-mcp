# Prompt‑Enhancer MCP – CLI (`mcp`)

The CLI provides a fast, one‑line way to optimise prompt drafts locally using your Ollama model.

## Installation

```bash
npm i -g prompt‑enhancer-mcp   # installs the `mcp` binary globally
```

The package ships a **bin** entry that points to `dist/bin/mcp.js`. After `npm run build` the compiled file will be generated automatically.

## Usage

### Pipe a draft into the CLI

```bash
echo "Write a concise summary of the project plan." | mcp
```

### Provide the draft as an argument

```bash
mcp --draft "I need a short, friendly reminder about my meeting tomorrow."
```

### Enable brainstorming mode

```bash
mcp --draft "Ideas for a weekend hackathon." --brainstorm
```

### Include an explanatory note

```bash
mcp --draft "Summarise the repository README." --explain
```

### Target a specific model format

```bash
mcp --draft "Generate a prompt for Claude." --target claude
```

### Override the Ollama model name

```bash
mcp --draft "Create a prompt for a coding assistant." --model llama3.1
```

## Options Overview

| Flag | Description |
|------|-------------|
| `-d, --draft <draft>` | Prompt draft string. If omitted, the CLI reads from **stdin**. |
| `-b, --brainstorm` | Enables multi‑persona brainstorming (via the `brainstorm` parameter). When omitted, `brainstorm` is left unset (not forced to `false`), so intent classification can still auto-enable it for ideation drafts unless `--no-auto-intent` is passed. |
| `-e, --explain` | Returns an extra one‑line summary of the changes made. |
| `-t, --target <model>` | Desired target model (`generic`, `claude`, `gpt4o`, `gemini`). |
| `-m, --model <ollama>` | Override the Ollama model used for optimisation. |
| `-u, --ollama-url <url>` | Override the Ollama base URL for this invocation (sets `OLLAMA_BASE_URL`). |
| `-H, --ollama-header <key=value>` | Extra header to send to Ollama, e.g. for an authenticating proxy (repeatable; merges into `OLLAMA_EXTRA_HEADERS`). |
| `-s, --session <id>` | Session ID to maintain conversation state across calls. |
| `--no-cot` | Disable automatic Chain-of-Thought injection (enabled by default). |
| `--no-guardrails` | Disable automatic negative-constraint injection (enabled by default). |
| `--no-auto-intent` | Disable automatic intent classification (web-search/artifact hints and auto-brainstorm), enabled by default. |
| `--no-auto-repair` | Disable automatic repair of lint findings (enabled by default). |
| `--stats` | Include token count and efficiency stats in the output. |
| `--engine <engine>` | LLM engine to use: `ollama` or `anthropic` (note: `sampling` is not available from the CLI). |
| `-h, --help` | Show help information (auto‑generated by **commander**). |

## Subcommands

### mcp lint

Lints a prompt for common issues without making any LLM calls. Checks for unresolved placeholders (`{{placeholder}}`), suspect acronym expansions (validated against the `.prompt-enhancer.json` glossary if present), and leaked meta-commentary.

**Usage:**
```bash
mcp lint [prompt]
```

**Options:**
| Flag | Description |
|------|-------------|
| `[prompt]` | Prompt text to lint. If omitted, the CLI reads from **stdin**. |
| `--draft <draft>` | Original draft text. When provided, enables draft-comparison rules (e.g. acronym expansion checks against the glossary). |
| `--context <context>` | Background context the prompt was built from. Helps lint interpret domain-specific terms. |

**Exit Codes:**
- `0` — No lint issues found.
- `1` — One or more lint issues detected.

**Example:**
```bash
mcp lint "Please summarize the document." --draft "I want a short summary"
```

### mcp score

Judge-grades a prompt on five dimensions: clarity, specificity, structure, guardrails, and token efficiency. Each dimension is scored 1–5 with a rationale. Optionally compare two prompts with the `--baseline` flag to see per-dimension deltas.

**Usage:**
```bash
mcp score [prompt]
```

**Options:**
| Flag | Description |
|------|-------------|
| `[prompt]` | Prompt to score. If omitted, the CLI reads from **stdin**. |
| `--baseline <prompt>` | Second prompt for comparison mode. When provided, score returns per-dimension deltas and a winner verdict. |
| `--engine <engine>` | LLM engine for scoring: `ollama` or `anthropic`. Defaults to preset or `ollama`. |
| `-m, --model <model>` | Override the judge model used for scoring. |

**Output:**
Returns a markdown table with scores on each dimension, per-dimension rationales, and an overall score.

**Example (single-mode):**
```bash
mcp score "Summarize the attached document in exactly 50 words."
```

**Example (comparison-mode):**
```bash
mcp score "Summarize the document." --baseline "Please provide a concise summary."
```

### mcp skills install

Installs the bundled `draft-prompt` Claude Code skill (an interactive
interview that assembles a `draft`/`context` pair and calls `optimize_prompt`
for you) so it's available as `/draft-prompt` in Claude Code.

**Usage:**
```bash
mcp skills install [--project]
```

**Options:**
| Flag | Description |
|------|-------------|
| `--project` | Install into `./.claude/skills/draft-prompt/` (current project) instead of `~/.claude/skills/draft-prompt/` (user-level, available in every project). |

Re-running the command always overwrites the destination file with the
bundled version — this is also how you pick up skill updates after
upgrading the package. There is no separate uninstall command; remove the
skill with `rm -rf ~/.claude/skills/draft-prompt` (or the project-scoped
path).

**Exit Codes:**
- `0` — Installed successfully.
- `2` — Install failed (e.g. permission error writing to the destination).

**Example:**
```bash
$ mcp skills install
Installed draft-prompt skill to /Users/you/.claude/skills/draft-prompt/SKILL.md
```

## Exit Codes

### mcp (root optimize command)

| Code | Meaning |
|------|---------|
| `0` | Success – optimized prompt(s) written to `stdout`. |
| `1` | Invalid usage (e.g., missing draft). |
| `2` | Internal error (network, Ollama failure, etc.). |

## Example Output

```text
$ echo "Give a detailed explanation of quantum entanglement." | mcp --explain
Optimized prompt: Explain quantum entanglement in ≤ 150 tokens, focusing on core concepts and avoiding jargon.

Explanation: Removed redundant phrasing and tightened the request to a concrete token budget.
```

## Development

After cloning the repo:

```bash
npm install          # installs dependencies (including commander & cosmiconfig)
npm run build        # compiles TypeScript to `dist/`
npm start            # runs the server (for API mode)
```

The CLI source lives at `src/bin/mcp.ts`. When you run `npm run build`, the compiled file appears at `dist/bin/mcp.js` and is exposed via the `bin` field in `package.json`.
